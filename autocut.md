# AI 음성 스토리 기반 자동 컷편집 MVP

## 프로젝트 개요

본 프로젝트는 사용자가 5분 내외 영상을 업로드하면 버튼 한 번으로 음성 기반 스토리 분석을 수행하고, 핵심 장면만 남긴 자동 컷편집 영상과 자동자막 결과물을 생성하는 AI 영상 편집 도구를 목표로 한다.

초기 버전에서는 복잡한 화면 인식보다 음성 스토리 분석 중심 구조를 채택하여 구현 난도를 낮추고 실사용성을 높인다.

---

## 핵심 사용자 경험

사용자는 복잡한 편집 과정을 거치지 않는다.

영상 업로드 후 **‘AI 초벌편집’ 버튼을 누르면 자동으로 핵심 구간만 남긴 편집 영상이 생성**된다.

최종 결과물에는 자동 자막이 포함되며, 사용자는 편집 결과를 다운로드하거나 추가 수정할 수 있다.

사용 흐름은 다음과 같다.

**영상 업로드 → AI 초벌편집 버튼 클릭 → 자동 컷편집 및 자막 생성 → MP4 결과물 다운로드**

---

## MVP 핵심 기능

초기 MVP에서는 기능을 최소화한다.

첫째, 음성 인식 기반 텍스트 추출 기능을 제공한다. 영상 속 음성을 텍스트로 변환하고 문장별 타임스탬프를 생성한다.

둘째, 음성 스토리 분석 기능을 제공한다. AI는 대화 흐름을 분석하여 중요 문장과 불필요한 구간을 판별한다.

셋째, 자동 컷편집 기능을 제공한다. 중요도가 낮은 구간을 제거하고 핵심 흐름 중심으로 재구성한다.

넷째, 자동자막 기능을 제공한다. 음성 인식 데이터를 활용하여 자막을 자동 삽입한다.

---

## 자동 컷편집 판단 기준

초기 버전에서는 AI가 감성 편집을 수행하지 않는다.

명확한 규칙 기반 편집 전략을 적용한다.

AI는 핵심 설명, 행동 설명, 결론, 감탄, 정보 전달 구간의 중요도를 높게 평가한다.

반대로 침묵, 반복 설명, 불필요한 군더더기 표현, “음…”, “어…” 같은 불필요 발화는 제거 대상으로 판단한다.

이를 통해 영상 길이를 줄이면서도 이야기 흐름은 유지하는 방향으로 편집한다.

---

## 자동자막 생성 구조

자동자막은 별도 기술을 추가하지 않고 음성 인식 데이터를 재사용한다.

텍스트와 타임스탬프 정보를 기반으로 문장 단위 자막을 생성한다.

초기 버전에서는 2줄 제한, 자동 줄바꿈, 화면 하단 배치 정도만 적용한다.

복잡한 키네틱 자막이나 디자인 자막은 제외한다.

---

## 시스템 처리 흐름

시스템은 매우 단순한 구조로 설계한다.

영상 업로드 후 음성 인식을 수행하고, AI가 문장 중요도를 평가한 뒤 편집 구간을 결정한다.

이후 영상 엔진이 자동으로 컷편집을 수행하고 자막을 삽입한 최종 영상을 출력한다.

구조는 다음과 같다.

**영상 업로드 → 음성 인식(STT) → 스토리 분석 → 컷편집 플랜 생성 → 자동 컷편집 → 자동자막 삽입 → 결과물 생성**

---

## 기술 구조

영상 처리 엔진은 FFmpeg 기반으로 구성한다.

AI는 실제 영상을 편집하는 것이 아니라 어떤 구간을 남길지 판단하는 역할을 수행한다.

즉, AI가 편집 플랜을 만들고 영상 엔진이 실제 렌더링을 담당하는 구조다.

AI는 뇌 역할을 하고, 영상 엔진은 손 역할을 담당한다.

---

## UI/UX 구조

초기 화면은 매우 단순하게 구성한다.

상단에는 영상 업로드 영역을 배치하고, 중앙에는 ‘AI 초벌편집’ 버튼 하나만 제공한다.

처리 완료 후 결과 영상 미리보기와 다운로드 버튼을 노출한다.

복잡한 편집 타임라인 UI는 초기 MVP에서 제외한다.

---

## 향후 확장 방향

초기 MVP 성공 이후 화면 인식 기반 편집 기능을 추가할 수 있다.

예를 들어 얼굴 표정, 제품 등장, 음식 완성 장면, 화면 변화 등을 분석하여 편집 품질을 높인다.

또한 쇼츠 자동 생성, 하이라이트 추출, 도메인별 편집 프리셋(요리·강의·브이로그 등) 기능도 확장 가능하다.

초기 목표는 ‘영상 업로드 후 버튼 한 번으로 usable한 초벌편집 영상 생성’에 집중한다.
